  1. [In this assignment, we use the DOM. Explain how this library can help us develop interactive web pages.]

    2. DOM provides a structured representation of the page and an API to manipulate it. Basically, without DOM, Javascript wouldn't be able to interact with page elements like buttons, tables, or dropdowns.

  2. [What is an event? Simply explain what happens when we add an event listener to an object.]

    3. An event makes it so the user can interact with the object (for eksample, clicking on it)

  3. [Sometimes we need to use an event object to get data from the event. Explain how this event object is structured, don't explain all functions or attributes. Try to explain, for example, how to retrieve data from a keyboard or mouse.]

    4. When an event occurs, the browser creates an event object that contains information about that spesific event. This object includes common properties like the event type and the element that triggered it, and event-spesific data as which key was pressed on the keyboard or where the mouse clicked. Using the event object allows our code to react based on the details of the user's action.

  4. [Explain the main difference between DOM id and DOM name. Can we have identical IDs?]

    5. The main difference of these two are: DOM id is meant to be unique in the entire HTML page, while DOM name is not required to be unique.
    And no, we can't have identical IDs on the same page, as only the first element will be selected while others get ignored.
